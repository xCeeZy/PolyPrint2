# АИС "PolyPrint"

Автоматизированная информационная система для учёта оборудования, клиентов и сервисных заявок.

## Описание проекта

PolyPrint — настольное WPF-приложение для компании, которая:
- Продаёт оборудование
- Ведёт клиентов
- Регистрирует сервисные заявки
- Фиксирует выполненные работы

## Технологии

- .NET Framework 4.7.2
- WPF (Windows Presentation Foundation)
- Entity Framework 5.0
- SQL Server / LocalDB

## Архитектура проекта

```

├── AppData/                 # Служебные классы
│   ├── AuthService.cs       # Авторизация
│   ├── NotificationService.cs # Уведомления

│   └── *GridItem.cs         # Классы-представления для DataGrid
├── Resources/               # Ресурсы приложения
│   └── Styles.xaml          # Общие стили
├── Model/                   # Модели Entity Framework
│   ├── Model1.edmx          # EDMX модель
│   └── *.cs                 # Классы моделей
├── View/
│   ├── Windows/             # Окна приложения
│   │   ├── LoginWindow      # Окно авторизации
│   │   ├── MainWindow       # Главное окно
│   │   ├── EditClientWindow # Редактирование клиента
│   │   └── EditUserWindow   # Редактирование пользователя

```

## Роли пользователей

| Роль | Права доступа |
|------|---------------|
| Администратор | Полный доступ ко всем функциям |
| Менеджер | Клиенты, оборудование, заказы, сервисные заявки |
| Мастер | Только сервисные заявки (назначенные ему) |

## Запуск проекта

1. Открыть решение в Visual Studio 2017 или выше
2. Восстановить пакеты NuGet
3. Выполнить скрипт `Database/TestData.sql` для создания тестовых данных
4. Убедиться, что строка подключения в App.config указывает на вашу БД
5. Запустить проект (F5)

## Стандарты кода

- Строгая типизация без `var` (кроме синтаксически неизбежных случаев)
- Без комментариев
- Код разделён на #region блоки
- Все имена в PascalCase
- XAML: только Grid, без Padding, Margin только внутри Grid
- Entity Framework с EDMX моделью
- Статический контекст: `App.context`

## Структура базы данных

- **Users**: пользователи системы
- **Clients**: клиенты компании
- **Equipment**: оборудование клиентов
- **Orders**: заказы
- **Order_Items**: позиции заказов
- **Service_Requests**: сервисные заявки
- **Works**: выполненные работы по заявкам
- **Parts**: запчасти
- **Used_Parts**: использованные запчасти

## Основной функционал

### Для менеджера:
- Добавление и редактирование клиентов
- Управление оборудованием
- Создание заказов
- Регистрация сервисных заявок

### Для мастера:
- Просмотр назначенных заявок
- Добавление выполненных работ
- Указание стоимости работ

### Для администратора:
- Все функции менеджера и мастера
- Управление пользователями
- Полный доступ ко всем данным
